# ask — Built-in SDK assistant

Built-in agent that answers questions about the MindStudio SDK. Invoked via `mindstudio ask "..."` (CLI), the `ask` MCP tool, or programmatically via `runAsk()`.

## Architecture

```
index.ts   — runAsk() core + cmdAsk() CLI wrapper
sse.ts     — SSE client for POST /_internal/v2/agent/chat
tools.ts   — 4 tool definitions + executeTool() dispatcher
prompt.ts  — system prompt builder (identity → reference → instructions)
types.ts   — Message, ToolCall, ToolDefinition, StreamEvent
```

### System prompt

Follows Anthropic's long-context guidance:
1. **Identity** (top — primacy): who the agent is
2. **Reference docs** (middle — bulk): llms.txt, model summary, connections, connector services
3. **Instructions** (bottom — recency): tools, response format

All reference sections wrapped in XML tags (`<sdk_reference>`, `<actions>`, `<models>`, etc.).

### Startup data

`buildSystemPrompt()` fetches live data in parallel before the first LLM call:
- `listModelsSummary()` — compact model catalog
- `listConnections()` — user's OAuth connections
- `listConnectors()` — available connector services
- `llms-content.ts` — bundled llms.txt (generated by codegen)

All fetches are wrapped in `Promise.allSettled` — the agent degrades gracefully if any fail.

### Tools

| Tool | Source | API call? |
|------|--------|-----------|
| `getActionDetails` | `stepMetadata` from generated code | No |
| `listModels` | `agent.listModelsSummary[ByType]()` | Yes |
| `getModelDetails` | `agent.listModels()` | Yes |
| `getConnectorDetails` | `agent.getConnector()` / `getConnectorAction()` | Yes |

### Agent loop

Simple tool-calling loop against `/_internal/v2/agent/chat`:
1. Send system prompt + messages + tool definitions
2. Stream SSE events, collect text + tool_use
3. If tool calls: execute in parallel, append results, loop back to 1
4. If no tool calls: return accumulated text

## Testing locally

```bash
npm run build
node dist/cli.js ask "generate an image with FLUX"
node dist/cli.js ask "what models support vision?"
node dist/cli.js ask "what connectors could I configure?"
```

Requires auth via `MINDSTUDIO_API_KEY` env var or `mindstudio login`.
